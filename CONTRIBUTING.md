# Contributing to VSOS Guard

Thank you for your interest in contributing!

## Reporting Bugs
1. Check [Issues](https://github.com/woshilaohei/vsos-guard/issues) first
2. Open a new issue with: description, steps to reproduce, expected vs actual, environment

## Suggesting Features
Open an issue with label `enhancement`. Describe the use case and why it matters.

## Submitting Changes
1. Fork the repo
2. Create a feature branch
3. Make changes + add tests
4. Open a Pull Request

## Code Principles
- Pure rule engine: no LLM calls, no network requests
- Zero false positives > zero missed attacks
- Every rule must have a reason and a suggestion
